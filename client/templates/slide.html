NPM (Node Package Manager)

Que es NodeJS?

Entorno de programación orientado a eventos basado en el lenguaje javascript,pensado para construir programas de red rápidos y escalables, con modelos de entrada/salida sin bloqueo que hacen liviano y eficiente el entorno. Perfecto para aplicaciones en tiempo real y de datos intensivos.

Orientado a eventos

Mientras en la programación secuencial (o estructurada) es el programador el que define cuál va a ser el flujo del programa, en la programación dirigida por eventos será el propio usuario quien los determine.

Lenguajes que usan el paradigma(estilo):

  • Java
  • VisualBasic
  • Javascript

Javascript

Es un lenguaje conocido por su uso en los navegadores, el cual permite una interación con el usuario, control del navegador, comunicación asiconcrónica y modificación del contenido que esta siendo mostrado.

Se clasifica como un lenguaje script basado en prototipos(estilo de programación orientada a objetos) y es multiparadigma.

Por qué NodeJS usa Javascript(JS) del lado del servidor?

"JavaScript has certain characteristics that make it very different than other dynamic languages, namely that it has no concept of threads. Its model of concurrency is completely based around events"

-Ryan Dahl-(creador de NodeJS)

NPM

npm es manejador de paquetes para javascript, y que trae por defecto NodeJS. Es el responsable de manejar las dependencias para Node y permite a los usuarios instalar apliaciones ya disponibles registradas en npm.

motor V8

Es el motor open-source escrito en C++ desarrollado por Google y utilizado en el navegador Google Chrome Es el encargado de compilar Javascript. Este motor se puede utilizar independientemente, incluso ser embebido en aplicaciones de C++.

Incluso se puede correr C++ dentro de una aplicación de nodeJS

Libuv

Libuv es una libreria escrita en C++, responsable de la asincronia de las operaciones I/O y el principal bucle de eventos (event-loop). Esta compuesta por un thread-pool de tamaño fijo desde el cual se alojan las operaciones de entrada y salida.

Apache vs NodeJS

Apache crea un nuevo hilo por cada conexión cliente-servidor, mientras que NodeJS solo uno y usa un hilo, y una cantidad de hilos fijo por cada por cada operación bloqueante (I/O).
-En un sistema UNIX usando nodeJS alrededor de 65.000 conexiones.(pero depende de varios factores)
